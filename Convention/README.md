## 깃 컨벤션

### 브랜치 종류
  - master: 최종 배포 브랜치
  - develop: 다음 출시 버전을 개발하는 브랜치(merge 대상 브랜치로 사용)
  - test: CI/CD 및 기타 테스트용 브랜치

### 커밋 브랜치
  - 개인 브랜치(영문 이름) -> Merge Request -> develop 브랜치 -> 개발 완료 -> master 브랜치

### 팀 커밋 컨벤션
  - <타입>[적용범위]: <설명>
    (ex: Feat[Server]: 로그인 api 추가)

  - 커밋 타입 <br />
